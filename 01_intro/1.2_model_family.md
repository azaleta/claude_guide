# 1.2 Claude 模型家族全景

Claude 不是一个单一模型，而是一个持续演进的**模型家族**。理解各代模型的特点，能帮助你在不同场景下做出正确选择。

---

## 模型命名规则

Claude 的模型命名遵循以下规则：

```
Claude [版本号] [系列名]
```

- **版本号**：如 3、3.5、4、4.5，数字越大代表代际越新
- **系列名**：
  - **Opus**：旗舰级，最强能力，适合复杂任务
  - **Sonnet**：平衡型，兼顾能力与速度
  - **Haiku**：轻量级，快速响应，低成本

例如：
- `Claude 3.5 Sonnet` = 第 3.5 代平衡型模型
- `Claude 4 Opus` = 第 4 代旗舰型模型

## 当前主力模型详解

### Claude 4.5 Opus

**定位**：旗舰模型，最强综合能力

| 特性 | 说明 |
|------|------|
| 上下文窗口 | 200K Token |
| 最大输出 | 8,192 Token |
| 视觉能力 | ✅ 支持图像分析 |
| 工具调用 | ✅ 支持 |
| Computer Use | ✅ 支持 |
| 适用场景 | 复杂推理、专业软件工程、高级 Agent |

**使用建议**：仅在需要最高质量输出时使用，成本较高。

---

### Claude 4.5 Sonnet

**定位**：最佳性价比，Agent 首选

| 特性 | 说明 |
|------|------|
| 上下文窗口 | 200K Token（1M 预览） |
| 最大输出 | 8,192 Token |
| 视觉能力 | ✅ 支持 |
| 工具调用 | ✅ 支持 |
| Computer Use | ✅ 支持 |
| 适用场景 | 日常编程、Agent 构建、复杂对话 |

**使用建议**：大多数场景的首选模型。

---

### Claude 4.5 Haiku

**定位**：快速响应，低成本

| 特性 | 说明 |
|------|------|
| 上下文窗口 | 200K Token |
| 最大输出 | 8,192 Token |
| 视觉能力 | ✅ 支持 |
| 工具调用 | ✅ 支持 |
| Computer Use | ✅ 支持 |
| 适用场景 | 简单任务、批量处理、成本敏感场景 |

**使用建议**：约为 Sonnet 90% 的能力，但只需 1/5 的成本。

## 成本计算示例

假设每月处理 100 万 Token（输入+输出），不同模型的成本对比：

| 模型 | 每百万 Token 估价* | 月成本估算 |
|------|-------------------|-----------|
| Claude 4.5 Haiku | ~$1 | ~$1 |
| Claude 4.5 Sonnet | ~$5 | ~$5 |
| Claude 4.5 Opus | ~$25 | ~$25 |

*价格为示意，实际以 [Anthropic 官网定价](https://www.anthropic.com/pricing) 为准。

## 已废弃的模型

以下模型已停止更新，但在某些 API 端点仍可访问：

- Claude 1.x 系列
- Claude 2.x 系列
- Claude 3.0 系列

**建议**：新项目应直接使用 Claude 4.x / 4.5.x 系列。

## 小结

- Claude 模型分为 Opus（旗舰）、Sonnet（平衡）、Haiku（轻量）三档
- 版本号越高，能力越强，但不一定需要最新最强的模型
- 2025 年末的主力模型是 Claude 4.5 系列
